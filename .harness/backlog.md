## 2026-04-26: 컨텍스트 압박률 지표 대체 방안 미결

- **백로그 이유**: 정확한 "세션 종료 시 컨텍스트 크기" 데이터가 DB에 없고, 현재 공식(`cacheRead/sessions/200K`)은 턴마다 누적되는 값이라 수백%가 나오는 근본 오류 — 올바른 지표를 결정하려면 DB 스키마 검토 또는 신규 데이터 수집 방안 확정이 선행되어야 함
- **할 것**: 아래 두 대안 중 하나를 선택하고 구현
  1. **원샷 성공률 카드**로 교체 — 정의·집계 쿼리 작성 후 카드 컴포넌트 교체
  2. **세션당 캐시 쓰기(M) 카드** 유지 — 공식을 `cacheWrite/sessions`(토큰→M 단위 환산)으로 단순 변경
- **필요한 것**: DB 스키마(`sessions`, `turns` 테이블) 재확인, 원샷 성공률을 쓰려면 "단일 턴 세션" 판별 기준 정의
- **이전 검토**: `cacheRead` 값이 세션 누적합임을 확인, 200K 나누기 방식은 무의미한 수치 생성 확인 / 세션당 캐시 쓰기(M) 표시와 원샷 성공률 두 가지 대안까지 좁혀진 상태
